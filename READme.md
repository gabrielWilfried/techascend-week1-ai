## What I Learned

* Python variables are dynamically typed, but adding type annotations makes code clearer and easier to maintain
* `enumerate()` is cleaner than manually tracking an index counter inside a loop
* Dictionaries are a powerful way to group related data, and `.items()` makes iterating over them readable
* Good functions do one thing only — separating logic from I/O makes code easier to test and reuse
* Writing a docstring with `Args` and `Returns` sections is a professional habit that pays off in team settings
